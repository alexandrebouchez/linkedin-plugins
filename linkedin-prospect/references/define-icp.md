# Define ICP

## Objectif

Créer ou mettre à jour `icp.md` dans le dossier de prospection. Ce fichier définit **qui** on cible et **pourquoi**, avec des critères de qualification clairs.

## Template cible

Voir `templates/icp.md.tpl` pour la structure de référence.

## Workflow

### Si des sources existent

Lire le `CLAUDE.md` du dossier et `product.md` (le produit informe directement les segments et le status quo). Pré-remplir un brouillon du template. Présenter à l'utilisateur pour correction/complétion, puis passer à la review.

### Si le fichier existe déjà

Lire `icp.md` et `product.md`, analyser proactivement les faiblesses (segments trop larges, critères vagues, trigger events manquants, incohérences avec le produit) et proposer des améliorations.

### Si on part de zéro

1. Lire `product.md` s'il existe pour contextualiser
2. Demander à l'utilisateur de décrire son client idéal
3. Analyser contre le template et combler les lacunes. Les zones critiques à couvrir : segments distincts avec angles différenciés, status quo concret, critères de qualification (bon vs mauvais prospect), exclusions avec raisons, trigger events mappés à des angles de message
4. Générer le fichier structuré

La cohérence avec `product.md` est essentielle : l'ICP découle du produit.

### Review

Valider les choix structurants : la cible prioritaire, les segments, les exclusions. Challenger ce qui semble trop large ou trop vague.

Ne pas terminer sans validation explicite.

## Output

Écrire `icp.md` dans le dossier de prospection.
