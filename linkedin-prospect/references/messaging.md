# Messaging — Recette de rédaction

Ce guide enseigne comment construire un bon message de prospection LinkedIn. Il ne définit pas le tone (c'est le rôle de `voice.md` du projet). Il donne la recette.

## Deux couches, une règle

| Couche | Source | Rôle |
|--------|--------|------|
| **Recette** (universelle) | Ce guide | Comment structurer un message qui cartonne |
| **Tone** (custom projet) | `voice.md` du projet | Comment ça sonne : vocabulaire, registre, exemples |

**Le tone du projet prime toujours.** Si `voice.md` contredit une règle de ce guide, le custom gagne. Les règles ici sont des defaults intelligents, pas des lois.

## Principe

Le prospect scanne un message LinkedIn en 8 secondes. Deux catégories : "message de vente" ou "message intéressant". Pas de troisième.

Ce qui fait basculer dans "intéressant" : le message lui apprend quelque chose sur son propre business, ou touche un enjeu qu'il n'a pas encore formulé. Ce qui fait basculer dans "vente" : le message parle du produit avant de parler de lui.

Le message doit sonner comme un pair qui parle à un pair, pas comme un vendeur qui pitche. Qui est ce pair exactement : c'est défini dans `voice.md`.

## Les 3 couches d'un bon message

### 1. L'accroche : leur business, pas le nôtre

Le premier tiers parle du prospect. Pas pour flatter ("j'ai adoré ton post"), mais pour montrer qu'on comprend son quotidien. Attaquer par un enjeu, une observation, un fait du marché.

L'accroche est spécifique à ce prospect ou à son segment. Elle formule un enjeu, pas un constat plat. "Je vois que tu fais du X" n'apporte rien. Formuler l'enjeu derrière le fait.

**Quand un trigger event est détecté** (funding, recrutement, changement réglementaire...), l'accroche s'ancre dedans. C'est le hook le plus puissant : timing + pertinence.

### 2. Le pitch : ce que ça fait, pas ce que c'est

On ne décrit pas le produit, on montre ce qu'il produit. Pas l'architecture technique, les inputs/outputs, le résultat concret. Le prospect s'en fiche du "comment".

Le pitch tient en 1-2 phrases. Les mots sont ceux du métier du prospect, pas du jargon interne.

### 3. La preuve + CTA : une stat vivante et une vraie question

Une stat par message max. Elle vit dans la phrase, elle ne la domine pas. "Les entreprises qui l'utilisent trouvent en moyenne X" plutôt que "x3 sur le volume de Y".

Le CTA est une vraie question qu'un humain poserait. Pas un script.

## Contraintes

- **Messages directs** : 28-35 mots recommandé, max 6 lignes. Densité max, chaque mot gagne sa place.
- **Invitations** : max 300 caractères (limite LinkedIn), pas de pitch, personnalisé
- **1 stat max** par message, intégrée dans la phrase
- **Jamais de lien** dans le M1
- **Le CTA** est une vraie question (pas "Pertinent ? Skip ou on en parle ?")
- **Unicité** : varier angles, stats, structures et CTAs entre les messages d'un batch. LinkedIn détecte le content hashing.

## Cadence follow-ups (recommandé)

- **J+4** : ping court rappelant le hook
- **J+10** : dernier message, micro-valeur de rappel
- **Max 2 follow-ups**

À valider avec l'utilisateur. La cadence exacte se configure dans Lemlist.

## Média (bonus)

L'option vidéo courte (Loom ou équivalent, 15-25s) peut booster les replies x2-3. Visage visible + screen (ex : screenshot résultat client). Proposer à l'utilisateur si le contexte s'y prête.

## Anti-patterns — Ce qui trahit un message généré

Le test simple : l'expéditeur dirait-il ça en face à face ? Si non, reformuler.

### Vocabulaire à surveiller

Ces mots sont des signaux fréquents de message généré. Ce ne sont pas des interdits absolus : si l'expéditeur les utilise naturellement (défini dans `voice.md`), ils sont légitimes.

| Mot/expression | Pourquoi c'est un signal |
|---------------|--------------------------|
| "solution" | Souvent corporate vide, mais légitime si c'est le mot du métier |
| "accompagner" | Peut sonner consultant, mais naturel dans certains contextes |
| "N'hésite pas" | Formule creuse, mais certains l'utilisent vraiment |
| "Au plaisir" | Signature robot dans 90% des cas |
| "Je reste disponible" | Faux signal d'accessibilité |
| "absolument", "incroyablement" | Superlatifs gratuits |
| "solution innovante" | Double signal corporate |
| "transformation digitale" | Buzzword sans substance |
| "révolutionner" | Promesse vide |

### Patterns structurels

| Pattern | Signal |
|---------|--------|
| Tiret long (em dash — ) | Signature IA |
| 3 points parfaitement équilibrés | Structure trop symétrique = robot |
| Tension-insight-preuve-CTA trop propre | Copywriting US, pas conversation |
| Phrases > 30 mots | Un humain tape plus court |
| Listes à puces dans un DM | Personne ne fait ça |
| Formule d'intro + compliment | SDR qui déroule un script |

### Patterns de contenu

| Pattern | Signal |
|---------|--------|
| Commencer par "Je" | Centré sur l'expéditeur |
| Flatterie d'intro | "Ton parcours impressionnant" = spam |
| Réciter ce que le prospect fait | Il le sait déjà |
| Pain sans bénéfice | "Et alors ?" |
| Bénéfice sans pain | "Trop beau" |
| Plusieurs questions | Charge cognitive |
| Lien dans le premier message | Spam flag LinkedIn |

### Filtre anti-générique

> Ce message pourrait-il être envoyé à 10 autres personnes sans changer un mot ?

Si oui, c'est du spam déguisé.

## 6 checks avant envoi

Chaque question doit avoir une réponse positive. Sinon, réécrire.

**1. Les faits se contredisent ?**
Chaque donnée doit être compatible. Si l'accroche mentionne un chiffre du prospect, le pitch ne peut pas annoncer moins sur la même dimension. Changer d'angle plutôt que créer un contresens.

**2. La personnalisation sert le pitch ?**
L'accroche personnalisée doit créer une tension que le pitch résout. Si l'info perso ne nourrit pas l'argument, c'est du bruit. La retirer ou en trouver une autre.

**3. Le déroulé est logique ?**
Phrase 1 → 2 → 3 : enchaînement causal. Si on peut intervertir deux phrases sans que ça change rien, le fil est cassé.

**4. L'angle est le bon pour CE prospect ?**
L'angle doit résonner avec la réalité du prospect, pas avec ce qu'on a envie de pitcher. Quel est SON problème ?

**5. Je parle dans le langage du prospect ?**
Chaque mot doit passer le test : le prospect utiliserait-il ce mot en réunion ?

**6. L'expéditeur dirait ça tel quel ?**
Relire à voix haute. Est-ce que ça sonne comme la personne décrite dans `voice.md` ? Si c'est bizarre, c'est faux.
